| Папка | Назначение |
|--------|------------|
| `src/` | Основное приложение (React + TS + MUI + Zustand) |
| `server/` | REST API на Express + SQLite (`better-sqlite3`) |
| `admin/` | Отдельная админ-панель (контент вопросов, импорт) |

**Стек основного приложения (`src/`):**

- **vite-plugin-pwa** (Service Worker, манифест)
- **Prism.js** (подсветка Kotlin в ответах)
- **Vitest** / **Playwright** (тесты; e2e — `npm run test:e2e`)

## Фронт на Vercel + API отдельно

Сайт на **Vercel** — это только статическая сборка. Запросы к вопросам идут на URL из **`VITE_API_URL`** (подставляется **при `npm run build`**).

1. В Vercel: **Project → Settings → Environment Variables** добавьте **`VITE_API_URL`** = полный URL вашего API, например `https://ВАШ-СЕРВИС.onrender.com/api` (без слэша в конце пути к `/questions` — только база `/api`).
2. Укажите для **Production** и при необходимости **Preview**.
3. Сделайте **Redeploy**, иначе в бандле останется старый URL.

Если **`VITE_API_URL` не задан**, в проде используется дефолтный Render из кода — это может **не совпадать** с тем API, куда вы заходите из админки, поэтому «в приложении не все вопросы».

Корень репозитория содержит **`vercel.json`** с SPA-rewrite на `index.html` для React Router.

## Деплой API и SQLite (Render и аналоги)

Вопросы хранятся в файле **`data.db`** (SQLite). На бесплатных веб-сервисах **файловая система контейнера сбрасывается при каждом redeploy**, если нет постоянного диска — данные из админки **теряются**. Демо-вопросы при пустой БД **не создаются**; пустой каталог — норма до импорта.

**Что сделать:**

1. **Постоянный диск** (например [Render Disk](https://render.com/docs/disks)): смонтировать каталог, например `/var/data`.
2. В настройках сервиса задать переменную окружения **одну из**:
   - `DATABASE_PATH=/var/data/ankor.db` — полный путь к файлу БД;
   - `SQLITE_DATA_DIR=/var/data` — каталог, внутри будет создан `data.db`.
3. Перезапустить сервис. В логах при старте видно строку `🗄️  DB: ...` — убедитесь, что путь на **смонтированном** диске, а не только внутри образа.

**Дополнительно:** перед redeploy без диска делайте резервную копию (экспорт из админки / копия `data.db` с сервера, если есть доступ).
